DATABASE STRUCTURE
==================

Database: PostgreSQL (Supabase)

moods table:
├── id (UUID, primary key)
├── user_id (UUID, foreign key → auth.users)
├── mood (TEXT) - one of: happy, sad, neutral, excited, anxious, calm
├── description (TEXT) - max 500 chars
├── image_url (TEXT, optional) - URL from storage
├── created_at (TIMESTAMP) - auto
└── updated_at (TIMESTAMP) - auto

Storage: mood_images bucket
└── Structure: {user_id}/{timestamp}_{filename}

SECURITY MODEL
==============

Authentication:
✅ Supabase Auth with email
✅ Email verification required
✅ Session management automatic

Authorization (RLS - Row Level Security):
✅ Users see ONLY their own entries
✅ Users can ONLY insert/update/delete their own data
✅ Storage: Users isolated by folder (user_id)

API CALLS FLOW
==============

Register → Supabase Auth.signUp()
    ↓
    Email confirmation needed

Login → Supabase Auth.signInWithPassword()
    ↓
    Session created + stored in browser

Dashboard → getMoodEntries() → supabase.from('moods').select()
    ↓
    RLS policy filters: where user_id = current_user
    ↓
    Only user's entries returned

Create Mood → createMoodEntry()
    ↓
    1. Upload image → supabase.storage.upload() (if provided)
    ↓
    2. Insert entry → supabase.from('moods').insert()
    ↓
    3. RLS: automatically sets user_id = current_user

Update Mood → updateMoodEntry()
    ↓
    Verify ownership + upload new image (if provided)
    ↓
    Update entry in database
    ↓
    RLS policy ensures only owner can update

Delete Mood → deleteMoodEntry()
    ↓
    Delete from database
    ↓
    RLS policy ensures only owner can delete
